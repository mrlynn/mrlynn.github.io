**Post 1 — Myth 1:**

Myth: "It's just VS Code with Copilot."

Reality: Cursor is a *fork*, not a plugin. Whole-codebase indexing and multi-file agents are built into the core — not bolted on.

One line: Familiar UI doesn't mean familiar architecture.

**Post 2 — Myth 2:**

Myth: "AI coding = unreviewed garbage in production."

Reality: Ask-before-terminal. Privacy Mode. SOC 2. You're still the reviewer — the tool just makes the first draft faster.

One line: Cursor makes you faster at writing code. It doesn't remove the need to read it.

**Post 3 — Myth 3:**

Myth: "It's just autocomplete."

Reality: Tab is the calculator app. Agent Mode plans, edits across files, runs tests, and waits for your approval.

One line: Tab is where you start. Agent Mode is where the product lives.

**Post 4 — Myth 4:**

Myth: "It's just another IDE on my laptop."

Reality: @Cursor in Slack. Tickets in Jira/Linear. MCP to your database. Bugbot on every PR.

One line: Cursor lives where your work already lives.

**Post 5 — Myth 5 + CTA:**

Myth: "Locked to one AI vendor."

Reality: Claude, GPT, Gemini, Grok, Composer — pick per task.

Try one boring experiment: Agent Mode → scoped task → read the diff → don't merge blind.
